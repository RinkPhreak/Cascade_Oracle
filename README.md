# Запуск CASCADE PRO (Инструкция)

Добро пожаловать в Cascade Oracle! В этом документе описано, как полностью развернуть и запустить систему (базы данных, бэкенд и фронтенд).

## 📋 Предварительные требования
Для работы адской машины на вашем компьютере должны быть установлены:
1. **Docker Desktop** (и запущен) — для баз данных (PostgreSQL, Redis) и, опционально, самого бэкенда.
2. **Node.js** (v18+) — для запуска фронтенд-части.

---

## 🛠️ Шаг 1: Запуск инфраструктуры и Бэкенда

Проект поставляется с полным `docker-compose.yml`, который поднимает все необходимые сервисы:
- Реляционную базу данных \`Postgres 16\`
- In-memory кэш и брокер очередей \`Redis 7\`
- Мониторинг очередей \`Asynqmon\`
- Бэкенд-сервис \`Cascade API\` (на `localhost:3000`)

1. Откройте терминал в корневой папке проекта (`cascade`).
2. Убедитесь, что Docker запущен.
3. Выполните команду для сборки и фонового запуска сервисов:
   ```bash
   docker-compose up --build -d
   ```
4. Убедиться, что всё работает, можно командой `docker-compose ps`.
*(UI для мониторинга асинхронных задач Asynqmon будет доступен по адресу: http://localhost:8080)*

---

## 🌐 Шаг 2: Запуск Фронтенда (React SPA)

Фронтенд находится в подпапке `frontend` и запускается через быстрый сборщик Vite. В файл `vite.config.ts` мы уже вшили прокси для автоматического перенаправления запросов с `/api` на бэкенд (`localhost:3000`).

1. Откройте **новый** терминал (или вкладку).
2. Перейдите в директорию фронтенда:
   ```bash
   cd frontend
   ```
3. Установите зависимости (если вы этого ещё не сделали):
   ```bash
   npm install
   ```
4. Запустите dev-сервер:
   ```bash
   npm run dev
   ```
5. Vite запустится и выдаст ссылку, обычно это: **http://localhost:5173/**

---

## 🔐 Шаг 3: Авторизация в системе

Перейдите в браузере по адресу фронтенда. Вас автоматически перенаправит на страницу `http://localhost:5173/login`.

Поскольку мы исправили сломанные ключи из `.env`, были сгенерированы следующие дефолтные доступы оператора:
- **Оператор (Login):** `admin`
- **Пароль (Password):** `admin`

После входа JWT-токен надежно фиксируется в zustand, а запросы обрабатываются с помощью TanStack Query. 

Удачного управления каскадами! 🚀
